👥 Player vs Player — Implementation Plan

    
Full PvP mode enabling two humans to play on the same device or across a network, with clocks, room codes, and post-game analysis.


    

      ✓ Phase 1 — Foundation (Done)
      
Core Chess Engine & Board

      
chess.js handles all move validation, legal move generation, check/checkmate/stalemate, castling, en passant, and promotion. The board renderer, coordinate system, eval bar, and piece display are fully production-ready.

      
chess.js@0.10.3 · FEN/PGN parsing · Unicode pieces · eval bar

    


    

      ✓ Phase 2 — Interactive Play (Done — This Release)
      
Manual Play & vs Computer

      
Click-to-move with legal hint dots, promotion picker, undo, board flip, horizontal move timeline, eval bar live update, export to Analyzer, and Stockfish-powered opponent at configurable depth levels 1–12.

      
handleSquareClick · legal-hint CSS · evalFen() · Stockfish Web Worker

    


    

      → Phase 3 — Local PvP (Next Sprint)
      
Two Players, One Device — Hot Seat Mode

      
    
        
  • Player name inputs (e.g. "Alice" and "Bob") shown in each player bar
    • 
        
  • Board auto-flips after each move so each player sees their own side naturally (toggle on/off)
    • 
        
  • Turn indicator: active player's bar pulses with an accent border
    • 
        
  • Chess clocks: configurable time controls — bullet (1+0), blitz (5+3), rapid (10+5), classical (30+0) — countdown displayed in MM:SS inside the player bar
    • 
        
  • On time expiration the opponent wins automatically with a clear banner
    • 
        
  • "Offer Draw" button: opponent must accept/decline; threefold repetition and 50-move rule detected automatically
    • 
        
  • "Resign" with one-click confirmation
    • 
        
  • End-of-game modal: winner, reason, move count, and a one-click "Analyze this game" button
    • 
      

      
setInterval clock · CSS pulse animation · draw offer state · checkmate/stalemate modal

    


    

      Phase 4 — Online Play (Planned)
      
Network PvP — Play a Friend

      
    
        
  • Generate a 6-character room code; share via link or copy button
    • 
        
  • WebSocket relay or WebRTC peer-to-peer connection — no server accounts needed
    • 
        
  • Both browsers see the board update in real time; only the active player can move
    • 
        
  • Reconnection grace period (60 s) if a player's connection drops
    • 
        
  • Spectator mode: additional people join the room as read-only viewers
    • 
        
  • Move sync uses UCI strings + FEN cross-check to prevent desync
    • 
      

      
WebSocket / WebRTC · room-code handshake · FEN sync · reconnect heartbeat

    


    

      Phase 5 — Profiles & History (Planned)
      
Player Profiles & Game Archive

      
    
        
  • Username + PIN login stored in IndexedDB (no server required for local)
    • 
        
  • Personal game history: date, opponent, result, opening, accuracy
    • 
        
  • Opening repertoire tracker: win rates as White/Black per ECO code
    • 
        
  • Elo/Glicko-2 rating updated after each ranked game
    • 
      

      
IndexedDB · Glicko-2 rating · ECO opening DB · PGN export

    


    

      Phase 6 — Tournament Mode (Planned)
      
Swiss / Round-Robin Tournaments

      
    
        
  • Create a tournament with 2–32 players; automatic Swiss pairings each round
    • 
        
  • Live leaderboard updated after each game
    • 
        
  • Tournament director controls: pause, adjudicate, add/remove players mid-event
    • 
      

      
Swiss pairing algorithm · bracket UI · tournament state machine

    


    

      
What it takes to ship Phase 3

      

        

          
1. Auto-flip on move

          Add flipOnMove flag; after move commit call manualFlipBoard().
        

        

          
2. Chess clock

          setInterval each second, decrement active player's time; display MM:SS in player bar; winner on 0.
        

        

          
3. End-game modal

          Semi-transparent overlay: winner name, reason, move count, "Analyze" and "Play Again" buttons.
        

        

          
4. Player name inputs

          Two text inputs in setup; stored in state, rendered in player bars.
